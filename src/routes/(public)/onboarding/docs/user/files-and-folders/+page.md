# Files and folders

## Browsing

**Home** lists the current folder’s contents. The sidebar and breadcrumbs reflect where you are: at the root you see only **Home**; inside a folder, the breadcrumb shows **Home** and the folder name (and deeper paths as you navigate).

Use the row actions on each item to manage files and folders. Shared and Trash views use the same list patterns with view-specific rules (for example, you cannot create folders from Shared or Trash).

## Labels and colors

Files and folders can carry a **color label** from the app’s fixed palette. Labels help you scan long lists and group related items visually.

## Pin and star

- **Pinned** items stay easy to find in the listing order the app applies.
- **Starred** items are marked for quick recognition.

Exact sort behavior follows the Home implementation; both flags are persisted with the item.

## Folders

Create folders from the **NEW** menu when you are in **Home** (not in Shared or Trash). Name validation strips risky characters and length limits apply on the server.
